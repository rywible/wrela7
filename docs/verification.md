### Discovered contract drift

- Syntax and HIR permit a vector of comparison tails even though the normative
  grammar forbids chained comparisons.
- The installed Unicode dependencies had advanced to Unicode 17 data while the
  language pins Unicode 16.0. Parser work must pin both XID and normalization
  tables to Unicode 16.
- Target/toolchain/test fixture codecs return fixed literal bytes rather than
  implementing their documented canonical formats.
- Target checks search for selected strings instead of parsing and
  cross-checking the complete package and native lock contracts.

These findings keep every row in `requirements-to-tests.md` at `Missing` or
`Contract only` until the real producers and consumers land.

An independent comparison to the normative layout algorithm found normal-line
dedents emitted as `NEWLINE, DEDENT` instead of the required
`DEDENT, NEWLINE` pairs. The scanner now looks ahead across blank/comment-only
lines, emits one exact pair per closed level, and preserves monotonic lossless
spans by attaching the first pair to the physical line ending and any further
pairs at its end. Checked-in regressions cover a second top-level declaration,
a two-level close, and EOF without a physical newline. Declaration and type
grammar is now complete. The syntax boundary is still not claimed complete
because statement, pattern, expression, and structured interpolation families
remain under implementation.
Cold-cache timing for the remaining slices is still pending; the input slice now
has both clean-target and warm evidence.

Manifest acquisition now has two explicit digest roles. Each exact raw root or
provider byte sequence is SHA-256 hashed before TOML decoding; semantic TOML is
then canonically encoded and only that canonical digest is compared with the
lockfile. An order-sensitive integration test proves all three hash-before-decode
boundaries in a one-package load while accepting a noncanonical but equivalent
TOML manifest.
